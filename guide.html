User Guide

A complete guide to using the OpenVideo Player — loading streams, configuring DRM, casting, and using diagnostics.

Getting Started

Open index.html in any modern browser. No installation needed — just serve the folder with any static file server:

python3 -m http.server 8000
# or
npx serve .

Then navigate to http://localhost:8000.

Loading a Stream

Enter a video URL in the Video URL input field and click Load Video. The player auto-detects the stream type:

URL EndingProtocolLibrary Used
.m3u8HLShls.js
.mpdDASHdash.js
Other (e.g. .mp4)NativeBrowser built-in
Tip: You can append query parameters to the URL (e.g., auth tokens). The player will pass them through to the stream request.

Skins and Themes

Player Skins

Use the Select Skin dropdown to change the player's visual style. Available skins: Modern, Light, Dark, Minimal, Cinema, Rounded, High Contrast, Retro, and Default.

Page Themes

The theme buttons in the header switch the entire page between Light, Dark, and Sepia color schemes.

DRM (Content Protection)

To play encrypted streams, configure the DRM license servers in the DRM Configuration panel:

  1. Enter the license server URL for your DRM system (Widevine, PlayReady, or FairPlay)
  2. For FairPlay, also enter the certificate URL
  3. Click Apply DRM Config
  4. Load your encrypted stream URL
DRM SystemSupported Browsers
WidevineChrome, Firefox, Edge, Android browsers
PlayReadyEdge, IE11
FairPlaySafari, iOS Safari
Tip: Most browsers support at least one DRM system. Chrome + Widevine is the most common combination for testing.
Note: DRM requires HTTPS in production. For local testing, localhost is treated as a secure context by most browsers.

QR Code (Stream URL Sharing)

Click the Show QR Code button to display a scannable QR code overlaid on the player. The QR code contains the current stream URL.

How to Use

  1. Load a stream in the player
  2. Click Show QR Code in the control panel
  3. Point your phone's camera at the QR code
  4. Your phone will show the stream URL — tap to open or copy it

This is useful for quickly transferring an ingest URL to a mobile device for testing or sharing with colleagues.

Casting (Chromecast & Fire TV)

When a cast-compatible device is detected on your network, a cast icon appears in the player's control bar.

  1. Click the cast icon
  2. Select your device from Chrome's native picker
  3. The stream will begin playing on the remote device
  4. Click the cast icon again to stop casting
Requirements: Casting requires Google Chrome. The cast device and browser must be on the same network. The stream URL must be accessible from the cast device (not localhost).

Fire TV

Fire TV devices can appear as cast targets if they have a DIAL-compatible receiver app installed (e.g., AirReceiver or Web Video Caster).

CMCD (Client Metrics)

The player automatically sends Common Media Client Data (CTA-5004) with each media request. This helps CDNs make better delivery decisions.

CMCD is enabled by default and transmits metrics like buffer length, bitrate, throughput, and session ID as query parameters on segment requests.

Diagnostics

Stats Overlay

Click the ⚙ Stats button on the video to show a real-time overlay with bitrate, resolution, codec, buffer length, dropped frames, and live latency.

Session Log

The Session Log panel shows a timestamped history of all player events: play, pause, seek, quality switches, errors, and recovery attempts.

Network Requests

The Network Requests panel logs each segment fetch with download speed, file name, and timing. Useful for spotting slow segments or CDN issues.

Buffer Health

The buffer bar shows how much video is buffered ahead. It turns orange below 3 seconds and red when critically low.

Troubleshooting

Stream won't play

  • Check the browser console for errors
  • Verify the URL is accessible (try opening it directly in a new tab)
  • For CORS issues, ensure your CDN sends proper Access-Control-Allow-Origin headers
  • For DRM streams, confirm the license server URL is correct and reachable

DRM errors

  • Ensure you're using HTTPS (or localhost)
  • Check that your browser supports the DRM system (e.g., Safari doesn't support Widevine)
  • Verify the license server is responding (check the Network tab in DevTools)

QR code shows wrong URL

  • The QR code shows whatever URL was last loaded. If you typed a new URL but didn't click "Load Video," the QR still shows the previous stream.

Cast icon not appearing

  • Casting only works in Google Chrome
  • Ensure the cast device is on the same network
  • Try reloading the page — the Cast SDK sometimes takes a moment to detect devices
